ConnectUni Web App — Implementation Report
==========================================


OVERVIEW
--------
ConnectUni is a full-stack mentorship platform connecting university students with alumni and
industry professionals. The frontend is a React + Vite + TypeScript SPA. This report documents
everything currently implemented.


TECH STACK
----------
Layer                   Technology
-----                   ----------
Framework               React 18 + Vite + TypeScript
Routing                 React Router v6
Server state            TanStack Query (React Query v5)
Auth state              Custom singleton hook (useAuth) — no external auth library
Styling (auth/onboard)  Custom au- CSS design system (auth.css)
Styling (dashboard)     Tailwind CSS + shadcn/ui components
Toasts                  Sonner

AUTHENTICATION — src/lib/api.ts + src/hooks/useAuth.ts
-------------------------------------------------------

API Helper (api)
  A typed fetch wrapper that handles JWT auth headers, token refresh, and error parsing.

  Methods:
    api.get<T>(path, params?)         — authenticated GET
    api.post<T>(path, body?)          — authenticated POST (JSON)
    api.postPublic<T>(path, body?)    — unauthenticated POST (no auth header)
    api.postForm<T>(path, formData)   — authenticated multipart POST
    api.loginForm<T>(path, email, password) — application/x-www-form-urlencoded login
    api.patch<T>, api.put<T>, api.delete<T> — standard REST methods
    tokens.set(access, refresh)       — store tokens in localStorage
    tokens.getAccess()                — read access token
    tokens.getRefresh()               — read refresh token
    tokens.clear()                    — remove both tokens

Auth Hook (useAuth)
  A singleton pub/sub pattern — a single _state object and _listeners set ensure all hook
  instances across the app stay in sync without needing context or Zustand.

  Boot sequence:
    On module load, reads the JWT from localStorage, decodes it client-side (no library —
    base64 + JSON.parse), extracts uid, email, and role claims, then fetches /profiles/me to
    hydrate the profile. All components using useAuth see loading: true until boot completes,
    preventing auth flashes.

  Exposed methods:
    Method                    Endpoint                    Description
    ------                    --------                    -----------
    signIn(email, password)   POST /auth/login            Form-encoded login, stores tokens, broadcasts state
    signUp(payload)           POST /auth/register         Registers new user (unauthenticated)
    signOut()                 POST /auth/logout           Revokes refresh token, clears state
    resetPassword(email)      POST /auth/forgot-password  Sends reset link
    refreshProfile()          GET /profiles/me            Re-fetches and broadcasts profile

  BackendRole values: STUDENT | ALUMNI | MENTOR | PROFESSIONAL | ADMIN


ONBOARDING DATA LAYER — src/hooks/useOnboarding.ts
---------------------------------------------------
TanStack Query hooks for the onboarding flow.

Key types:
  StudentProfile       — university_name, course_title, year_of_study, expected_graduation
  AlumniProfile        — university_name, course_completed, graduation_year, certificate_url
  ProfessionalProfile  — job_title, company, industry_sector, years_of_experience, linkedin_url
  MentorshipPreference — is_mentor, is_mentee, areas_of_interest[], availability_hours_per_week,
                         preferred_format
  FullProfile          — union of all above + verification_status, is_verified

Helper functions:
  hasRoleProfile(fp)       — returns true if the user's role-specific sub-profile exists
  getDashboardForRole(role) — maps role to /dashboard, /alumni-dashboard, or /admin

Mutations (all invalidate ['full-profile'] on success):
  useCreateStudentProfile()       → POST /profile/student
  useCreateAlumniProfile()        → POST /profile/alumni (multipart, supports certificate upload)
  useCreateProfessionalProfile()  → POST /profile/professional
  useSetMentorshipPreferences()   → POST /profile/mentorship/preferences
  useVerifyEmail()                → GET /auth/verify-email?token=


ROUTE GUARDS
------------

ProtectedRoute (src/components/layouts/ProtectedRoute.tsx)
  Wraps authenticated routes. Shows a skeleton loading state during auth resolution.
  Redirects unauthenticated users to /login (preserving `from` location for post-login redirect).
  Supports optional allowedRoles[] — users with a wrong role are sent to their own dashboard.

GuestRoute (src/components/layouts/GuestRoute.tsx)
  Wraps public-only routes. Returns null during auth load (prevents flash of login page for
  logged-in users). Authenticated users are redirected to their role dashboard via dashboardMap.
  Applied to: /, /login, /signup, /forgot-password.

OnboardingGuard (src/components/layouts/OnboardingGuard.tsx)
  Wraps dashboard routes (sits inside ProtectedRoute). Fetches FullProfile and checks completion:
    1. If no role profile        → redirect /onboarding/profile
    2. If no mentorship prefs    → redirect /onboarding/mentorship
    3. Otherwise                 → render children
  Waits for both isLoading AND isFetching before rendering, preventing stale-cache redirect
  loops immediately after onboarding submission.

COMPLETE USER JOURNEY
---------------------

New user lands on /
    → If logged in         → redirected to dashboard (GuestRoute)
    → If not               → sees LandingPage

Signs up at /signup
    → 3-step form: credentials → role → profile info
    → POST /auth/register
    → Redirected to /auth/check-email

Clicks verification link (email)
    → GET /auth/verify-email?token=...
    → Success → "You're in." → sign in link

Signs in at /login
    → POST /auth/login (form-encoded)
    → JWT stored, profile fetched
    → If no role profile       → /onboarding/profile
    → If no mentorship prefs   → /onboarding/mentorship
    → Otherwise                → role dashboard

Completes onboarding (2 steps)
    → Step 1: role-specific profile (student / alumni / professional)
    → Step 2: mentorship preferences
    → Navigated to dashboard

Uses dashboard features
    → All routes guarded by ProtectedRoute + OnboardingGuard
    → Incomplete onboarding mid-session → redirected back

Signs out
    → POST /auth/logout (refresh token revoked)
    → Tokens cleared, state broadcast to all hook instances
    → Next navigation to / or /login → GuestRoute passes through
